06 · 设计理念

设计说明

作为 AI 产品经理的设计决策

🎯 产品定位

  • • 目标用户：轻症患者、慢病随访者和体检人群
  • • 核心价值：多智能体协作降低误诊率，加速诊断流程
  • • 差异化：人类在环验证 + 多模态诊断 + RAG 知识库
  • • 技术壁垒：LangGraph 编排 + 置信度路由 + 混合检索

🧩 Agent 决策逻辑

  • • 有图片 → Vision Agent（根据图片类型路由）
  • • 医学知识 → RAG Agent（Qdrant 向量检索）
  • • 最新资讯 → Web Search Agent（Tavily + PubMed）
  • • 低置信度 → Web Search Fallback（Confidence < 85%）
  • • Vision 结果 → Human-in-the-Loop（强制审核）

🎨 设计系统决策

  • • 主色 Teal：医疗行业的信任感 + 科技感
  • • 强调色 Coral：风险/警告/需要注意的结果
  • • 字体 Outfit：现代几何感，在医疗严肃性中注入亲和力
  • • 圆角设计：降低医疗产品的冷峻感

🔒 安全与合规

  • • Input Guardrails：过滤不当输入
  • • Output Guardrails：确保输出安全合规
  • • Human-in-the-Loop：所有影像诊断均需人工审核
  • • 置信度阈值：低置信度结果自动触发 Web Search
